# COMBATUS OS — GLOSSARY

## 1. PURPOSE
Defines core system terms for consistent interpretation.

Prevents ambiguity in execution.

---

## 2. CORE TERMS

### Reality Layer
The filter that removes interpretation and keeps only verifiable facts.

### Decision Engine
System that selects exactly one executable action.

### Execution Layer
System that performs the selected action immediately.

### Micro-Step
Smallest possible physical action executable in ≤2 minutes.

### Reduction
Process of simplifying complexity until one action remains.

### Constraint
Hard limit that prevents system expansion or ambiguity.

---

## 3. SYSTEM TERMS

### Active Goal
The only currently processed objective.

### Blocker
Any condition that prevents execution of a micro-step.

### Failure State
Condition where no valid action can be derived.

### Feedback Loop
System mechanism that evaluates execution outcomes and adjusts structure.

---

## 4. RULE
If a term is not in this glossary:
→ it is not valid for system operation
