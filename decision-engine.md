# COMBATUS OS — DECISION ENGINE

## 1. PURPOSE
The Decision Engine converts input into a single executable action.

No multiple outputs. No parallel decisions.

---

## 2. INPUT FILTER
Input is reduced by removing:
- noise
- assumptions
- emotional interpretation
- unnecessary context

Only factual core remains.

---

## 3. CLARITY GATE
If the goal is unclear:
→ stop execution
→ reduce problem until one sentence remains
→ define single objective

---

## 4. PRIORITY LOGIC
Priority order:
1. irreversible impact
2. time critical
3. goal progress
4. emotional signal (only as indicator)

---

## 5. REDUCTION RULE
Any complex problem must be reduced to:
- 1 goal
- 1 action path
- 1 micro-step

No exceptions.

---

## 6. OUTPUT FORMAT
The engine always outputs:

→ 1 goal
→ 1 next action (≤2 minutes)

---

## 7. FAILURE MODE
If no decision can be made:
→ reduce scope further
→ remove abstraction
→ convert into physical action
