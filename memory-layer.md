# COMBATUS OS — MEMORY LAYER

## 1. PURPOSE
The Memory Layer stores only stable, recurring system patterns.

No short-term thoughts. No emotional logs.

---

## 2. STORAGE RULE
Only store information if:
- it repeats across multiple executions
- it changes decision quality
- it affects system stability

---

## 3. MEMORY TYPES

### 3.1 Pattern Memory
Recurring behaviors or failures

### 3.2 Structural Memory
System design improvements

### 3.3 Execution Memory
What consistently leads to successful action

---

## 4. NON-STORAGE RULE
Do NOT store:
- single events
- emotional reactions
- temporary confusion
- unverified assumptions

---

## 5. UPDATE RULE
Memory can only be updated when:
→ a pattern is confirmed
→ system behavior changes measurably

---

## 6. SYSTEM ROLE
Memory Layer = long-term compression of execution intelligence
