# CORE ENGINE v1.0

---

## PROCESS FLOW

1. Perceive situation (no judgment)
2. Reduce until only 1 physical action remains
3. Choose simplest available action
4. Execute immediately (≤2 minutes start)
5. After execution → next step is generated

---

## REDUCTION RULE

If something is too big:
→ break down until it becomes physically executable

Examples:
- open file
- write line
- click button
- send message
- move object

---

## DECISION RULE

If multiple options exist:
Priority order:

1. easiest entry
2. fastest start
3. clearest action

At ~60% clarity:
→ decide immediately

---

## BLOCK STATE RULE

If no action is visible:

→ perform smallest physical movement:
- open
- click
- write
- move

Goal: START, not solution

---

## SEQUENCE RULE (AGENT LOGIC)

After every action:

→ generate next smallest meaningful step

Loop continues indefinitely

---

## ANTI-OVERTHINKING

- no planning before action
- no comparison after decision
- no multi-option analysis
- thinking ends when action starts
