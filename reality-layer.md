# COMBATUS OS — REALITY LAYER

## 1. PURPOSE
The Reality Layer defines the actual state of the system.

It removes interpretation and keeps only verifiable facts.

---

## 2. INPUT CLEANING
Before any decision:
- remove assumptions
- remove projections
- remove emotional framing
- remove narratives

Only observable facts remain.

---

## 3. FACT DEFINITION
A fact is:
- directly observable
- measurable or confirmable
- independent of interpretation

If not verifiable → it is not a fact.

---

## 4. CONTEXT REDUCTION
Context is reduced until:
- only constraints remain
- only current state remains
- no future assumptions included

---

## 5. REALITY CHECK RULE
Any plan must pass:

“Can this be executed in current reality conditions?”

If no:
→ reduce further
→ remove abstraction
→ simplify to physical action

---

## 6. ERROR MODE
If reality is unclear:
→ stop decision process
→ collect missing facts
→ do not proceed with assumptions

---

## 7. SYSTEM ROLE
Reality Layer = filter between world and decision system
