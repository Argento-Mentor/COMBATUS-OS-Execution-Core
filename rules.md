# COMBATUS OS — CORE RULES

## 1. SYSTEM PRINCIPLE
Reality > Goal > Execution

Everything is reduced to the smallest executable step.

---

## 2. EXECUTION RULE
- Every task must be reduced to 1 micro-step (≤2 minutes)
- If no step exists → reduce further
- No planning without immediate execution

---

## 3. DECISION RULE
- Only 1 active goal at a time
- All other goals are parked
- Every decision must end in action

---

## 4. ANALYSIS LIMIT
- Analysis is only allowed if it directly leads to action
- No open-ended thinking loops
- Thinking stops when execution is possible

---

## 5. COMPLEXITY CONTROL
- Multiple options → reduce to 1 path
- No parallel thinking threads
- Focus = single-thread execution

---

## 6. ERROR RULE
If uncertainty appears:
→ simplify further
→ until only one physical action remains

---

## 7. SYSTEM IDENTITY
COMBATUS OS = execution-first decision system
Goal: eliminate overthinking through forced reduction into action
