# COMBATUS OS — EXECUTION LAYER

## 1. PURPOSE
The Execution Layer transforms decisions into immediate physical action.

No planning. No delay. No reinterpretation.

---

## 2. EXECUTION RULE
- Only 1 action is executed at a time
- Action must be physically performable
- Execution time limit: ≤ 2 minutes

---

## 3. MICRO-STEP PRINCIPLE
Every action must be:
- smallest possible unit
- directly executable
- independent (no dependencies)

---

## 4. EXECUTION LOOP
1. receive decision
2. reduce to micro-step
3. execute immediately
4. return to idle state

---

## 5. BLOCKERS HANDLING
If execution is blocked:
→ reduce action further
→ remove complexity
→ switch to simpler physical step

---

## 6. NO-THINK RULE
During execution:
- no re-evaluation
- no re-planning
- no alternative generation

---

## 7. SYSTEM OUTPUT
Execution always ends with:
→ done / not done state only
